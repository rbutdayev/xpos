## Executive Summary

This document defines the complete dashboard structure for a **multi-tenant point-of-sale (POS) system** with:

- **8 distinct user roles** with different access levels
- **5 optional modules** (Services, Rentals, Shop, Loyalty, Discounts)
- **4 integrations** (SMS, Telegram, Fiscal Printer, Loyalty)
- **Multi-tenant architecture** with strict account_id isolation
- **Dynamic conditional rendering** based on roles and enabled modules

**Key Principles:**
1. **Security First:** Never show data users shouldn't access
2. **Module Awareness:** Hide all features when module is disabled
3. **Role-Based Views:** Each role sees only relevant information
4. **Performance:** Load only necessary data for each role
5. **Clarity:** Clear visual hierarchy and actionable insights

## System Context

### Multi-Tenancy
- Each **Account** is a separate business (tenant)
- All data queries MUST filter by `account_id`
- Users belong to one account and can only see their account's data
- Cross-tenant data access is impossible via global scopes

### Role-Based Access Control (RBAC)

| Role | Authority | Primary Focus | Dashboard Access Level |
|------|-----------|---------------|------------------------|
| **account_owner** | Highest | Business overview, strategy | Full dashboard with all financial data |
| **admin** | Highest | System management | Full dashboard with all financial data |
| **accountant** | High | Financial health, compliance | Financial metrics, expenses, reports |
| **branch_manager** | Medium-High | Branch operations | Branch-specific metrics, local reports |
| **warehouse_manager** | Medium-High | Inventory, suppliers | Inventory metrics, stock alerts |
| **sales_staff** | Medium | Sales, customers, services | Sales metrics, customer data, service queue |
| **tailor** | Medium | Service management | Service queue, customer items |
| **cashier** | Low | POS operations only | Minimal dashboard, quick actions only |

### Module System

| Module | Default State | Purpose | Dashboard Impact |
|--------|---------------|---------|------------------|
| **Services** | Enabled | Repair/service management | Service KPIs, pending services, service revenue |
| **Rentals** | Enabled | Equipment rental | Rental KPIs, active rentals, overdue items |
| **Shop** | Disabled | E-commerce platform | Online orders, pending orders alert |
| **Loyalty** | Enabled | Customer rewards | Loyalty points stats, active members |
| **Discounts** | Enabled | Product promotions | Discount usage stats, revenue impact |

#### Data Requirements

**Must Load:**
- Inventory health metrics (total products, stock value, turnover)
- Stock status breakdown (healthy, low, out, negative)
- Stock by unit for selected warehouse
- Supplier insights (active suppliers, pending payments, goods receipts)
- Stock movement statistics (inbound, outbound, transfers, returns)
- Stock value trend (30 days)
- Category distribution
- Stock movement chart data
- Top suppliers by volume
- Low stock products (all)
- Pending goods receipts (all)
- Recent stock movements (last 50)

**Warehouse Filter:**
- All metrics should respect selected warehouse
- Default to user's default warehouse
- Allow "All Warehouses" view

**NO Access To:**
- Financial detailed reports (only see stock value)
- Sales data (only stock movements)
- Customer credit information
- User management

#### Data Requirements

**Must Load:**
- Personal performance metrics (my sales, my services, customers served)
- Service queue status (pending, in-progress, completed)
- Customer insights (active, new, repeat, retention)
- Rental status (if enabled) - user's rentals only
- Stock availability alerts (read-only)
- 7-day sales trend for current user
- Sales by category
- Pending services assigned to user
- Recent sales by user (last 10)
- Customers with overdue payments (for follow-up)

**Filtering:**
- All sales/service data should be filtered to current user
- Can see all customers (not just their own)
- Stock data is read-only for awareness

**NO Access To:**
- Overall financial metrics (revenue, expenses, profit)
- Other users' sales performance
- Stock value or cost prices (only sale prices)
- Supplier information
- System settings

- Branch daily sales (7 days)
- Staff performance comparison
- Peak hours analysis

- Today's sales activity

- All data should be filtered to current branch
- Cannot see other branches' data unless account_owner
- Can see branch-assigned staff performance

- Can manage local staff (not create/delete, only assign/schedule)
- Cannot see account-level financial details
- Cannot modify prices or products (read-only)

## KPI Definitions & Calculations

### Financial KPIs

#### 1. Total Revenue (This Month)
**Definition:** Sum of all sales revenue for the current month
**Calculation:**
```sql
SELECT SUM(total)
FROM sales
WHERE account_id = :account_id
  AND DATE(created_at) >= :month_start
  AND DATE(created_at) <= :month_end
  AND status != 'cancelled'
```

**Additional Modules:**
```sql
-- If services_module_enabled:
+ (SELECT SUM(total) FROM tailor_services WHERE ...)

-- If rent_module_enabled:
+ (SELECT SUM(total) FROM rentals WHERE ...)
```

**Growth Calculation:**
```sql
growth_percent = ((current_month - last_month) / last_month) * 100
```

---

#### 2. Total Expenses (This Month)
**Definition:** Sum of all business expenses for the current month
**Calculation:**
```sql
SELECT SUM(amount)
FROM expenses
WHERE account_id = :account_id
  AND DATE(date) >= :month_start
  AND DATE(date) <= :month_end
```

**Growth Calculation:** Same as revenue

---

#### 3. Net Profit (This Month)
**Definition:** Total Revenue - Total Expenses
**Calculation:**
```sql
net_profit = total_revenue - total_expenses
```

**Profit Margin:**
```sql
profit_margin = (net_profit / total_revenue) * 100
```

---

#### 4. Pending Payments
**Definition:** Total outstanding customer credit across all customers
**Calculation:**
```sql
SELECT SUM(credit_amount)
FROM customers
WHERE account_id = :account_id
  AND credit_amount > 0
```

**Count:**
```sql
SELECT COUNT(*)
FROM customers
WHERE account_id = :account_id
  AND credit_amount > 0
```

---

### Operational KPIs

#### 5. Active Customers
**Definition:** Customers who made at least one purchase in the last 90 days
**Calculation:**
```sql
SELECT COUNT(DISTINCT customer_id)
FROM sales
WHERE account_id = :account_id
  AND created_at >= NOW() - INTERVAL 90 DAY
```

---

#### 6. Products Count
**Definition:** Total number of unique products (active only)
**Calculation:**
```sql
SELECT COUNT(*)
FROM products
WHERE account_id = :account_id
  AND status = 'active'
```

---

#### 7. Total Stock Value
**Definition:** Sum of (quantity Ã— price) for all products in inventory
**Calculation (Cost Basis):**
```sql
SELECT SUM(ps.quantity * p.cost_price)
FROM product_stocks ps
JOIN products p ON ps.product_id = p.id
WHERE ps.account_id = :account_id
  [AND ps.warehouse_id = :warehouse_id if filtered]
```

**Calculation (Sale Value):**
```sql
SELECT SUM(ps.quantity * p.sale_price)
FROM product_stocks ps
JOIN products p ON ps.product_id = p.id
WHERE ps.account_id = :account_id
```

**Potential Profit:**
```sql
potential_profit = stock_value_sale - stock_value_cost
```

---

#### 8. Stock Turnover Ratio
**Definition:** How many times inventory is sold and replaced over a period
**Calculation:**
```sql
-- Cost of goods sold (this month)
cogs = SELECT SUM(si.quantity * si.cost_price)
       FROM sale_items si
       WHERE ...

-- Average inventory value (beginning + ending / 2)
avg_inventory = (beginning_stock_value + ending_stock_value) / 2

-- Turnover ratio (annualized)
turnover_ratio = (cogs / avg_inventory) * 12
```

**Interpretation:**
- **< 2**: Slow-moving inventory
- **2-4**: Normal turnover
- **4-6**: Good turnover
- **> 6**: Fast-moving inventory

## Data Access & Security

### Multi-Tenant Data Isolation

**Every Query MUST Include:**
```sql
WHERE account_id = :current_user_account_id
```

**Enforced By:**
- Global scope on all models using `BelongsToAccount` trait
- Middleware verification
- Gate authorization

**Example Safe Query:**
```php
// Automatic account filtering via global scope
$sales = Sale::where('created_at', '>=', $monthStart)
    ->where('created_at', '<=', $monthEnd)
    ->sum('total');

// Or explicit (for clarity)
$sales = Sale::byAccount(auth()->user()->account_id)
    ->where('created_at', '>=', $monthStart)
    ->sum('total');
```

**NEVER Do This:**
```php
// âŒ WRONG - Missing account filter
$sales = Sale::sum('total');

// âŒ WRONG - Using hard-coded account_id
$sales = Sale::where('account_id', 1)->sum('total');
```

---

### Role-Based Data Filtering

#### Controller Level
```php
public function index(Request $request)
{
    // Always authorize first
    Gate::authorize('access-dashboard');

    $user = auth()->user();
    $account = $user->account;

    // Load only what this role can see
    $data = match($user->role) {
        'account_owner', 'admin' => $this->getFullDashboard($account),
        'accountant' => $this->getFinancialDashboard($account),
        'warehouse_manager' => $this->getInventoryDashboard($account, $user),
        'sales_staff' => $this->getSalesDashboard($account, $user),
        'tailor' => $this->getServiceDashboard($account, $user),
        'branch_manager' => $this->getBranchDashboard($account, $user),
        'cashier' => $this->getCashierDashboard($user),
        default => abort(403)
    };

    return Inertia::render('Dashboard', $data);
}
```

#### User-Specific Filtering (Sales Staff, Tailor, Cashier)
```php
private function getSalesDashboard($account, $user)
{
    // Filter to user's own data
    return [
        'my_sales' => Sale::byAccount($account->id)
            ->where('created_by', $user->id)
            ->whereBetween('created_at', [$monthStart, $monthEnd])
            ->count(),

        'my_services' => TailorService::byAccount($account->id)
            ->where('assigned_to', $user->id)
            ->where('status', 'pending')
            ->count(),

        // Can see all customers (not filtered)
        'active_customers' => Customer::byAccount($account->id)
            ->where('status', 'active')
            ->count(),

        // Can see stock alerts (read-only)
        'low_stock_count' => $this->getLowStockCount($account),
    ];
}
```

#### Branch-Specific Filtering
```php
private function getBranchDashboard($account, $user)
{
    $branchId = $user->branch_id;

    return [
        'branch_revenue' => Sale::byAccount($account->id)
            ->where('branch_id', $branchId)
            ->whereBetween('created_at', [$monthStart, $monthEnd])
            ->sum('total'),

        'branch_staff' => User::byAccount($account->id)
            ->where('branch_id', $branchId)
            ->where('status', 'active')
            ->count(),
    ];
}
```

### Module-Based Data Loading

```php
private function getFullDashboard($account)
{
    $data = [
        // Core metrics (always loaded)
        'revenue' => $this->calculateRevenue($account),
        'expenses' => $this->calculateExpenses($account),
        // ...
    ];

    // Conditional loading based on modules
    if ($account->services_module_enabled) {
        $data['service_metrics'] = $this->getServiceMetrics($account);
    }

    if ($account->rent_module_enabled) {
        $data['rental_metrics'] = $this->getRentalMetrics($account);
    }

    if ($account->shop_enabled) {
        $data['online_orders'] = $this->getOnlineOrderMetrics($account);
    }

    if ($account->loyalty_module_enabled) {
        $data['loyalty_stats'] = $this->getLoyaltyStats($account);
    }

    return $data;
}
```
